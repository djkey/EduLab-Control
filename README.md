# Дистанционное управление классом

Приложение для уроков:
- режим **Учитель (сервер)**;
- режим **Ученик (клиент)**;
- первичная регистрация клиентов через облачный MQTT;
- отправка команды **открыть ссылку** всем подключённым ученикам;
- подпись команд через HMAC (проверка валидности на клиентах).

## Стек
- Python 3.10+
- PyQt5
- paho-mqtt

## Быстрый старт
1. Клонировать проект.
2. Создать и активировать виртуальное окружение.
3. Установить зависимости:

```powershell
pip install -r requirements.txt
```

4. Запуск:

```powershell
python main.py
```

## Первый запуск
- Выберите роль:
  - **Учитель**: локально генерируется и сохраняется ключ (в `config.json`).
  - **Ученик**: публикует анонс в облачную очередь, затем получает ключ от учителя при добавлении.

## Как отправить ссылку на уроке
1. Запустить учительское приложение.
2. На клиентах запустить режим ученика.
3. В окне учителя открыть плитку **Передача** → **Открыть ссылку**.
4. Ввести URL и отправить.

## Важные файлы
- `main.py` — точка входа
- `ui_server.py` — окно учителя
- `ui_client.py` — окно ученика
- `mqtt_handler.py` — работа с MQTT
- `crypto_utils.py` — подпись/проверка команд
- `config.py` — конфиг и топики

## Перенос на другой ПК
1. Скопировать/клонировать репозиторий.
2. Установить Python и зависимости.
3. Запустить `python main.py`.
4. Выбрать роль устройства на первом запуске.

## Примечание по безопасности
Сейчас команды подписываются HMAC, а клиент проверяет подпись и срок жизни сообщения.
Для production-релиза рекомендуется добавить TLS (MQTTS), ротацию ключей и белый список клиентов.
